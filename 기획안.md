### **앱 기획안: 로컬리 (Locally) v1.1**

**1. 앱 컨셉 및 목표**

- **컨셉:** 빅데이터와 현지인 커뮤니티를 결합하여, 여행자에게 광고성 정보가 아닌 실제 현지인들이 즐겨 찾는 장소와 경험을 추천하는 개인 맞춤형 여행 가이드 앱.
- **목표:** 여행자들이 방문하는 도시의 진정한 매력을 느끼고, 현지 문화에 더 깊이 스며들 수 있도록 돕는다.

**2. 타겟 유저**

- 패키지 여행보다 자유여행을 선호하는 20-40대 여행자.
- SNS의 유명 맛집/관광지보다 새롭고 독특한 경험을 추구하는 여행자.
- 여행지의 문화와 일상을 깊이 있게 체험하고 싶은 '경험 중심' 여행자.

**3. 핵심 성공 지표 (KPIs)**

- **활성 사용자:** 주간 활성 사용자 수 (WAU)
- **콘텐츠 기여도:** 사용자 1인당 주간 평균 스팟 등록 및 리뷰 수
- **사용자 만족도:** 재방문율, 스토어 평점, AI 추천 코스 채택률

**4. 핵심 기능 (Features)**

- **① 현지인 추천 스팟 (Local's Pick):**

  - 현지인 사용자들이 직접 자신의 단골 가게, 숨겨진 명소 등을 등록하고 '이야기'가 담긴 리뷰 작성.
  - **동기부여 시스템:** 우수 기여자 뱃지, 포인트 제도 등 게임화(Gamification) 요소를 도입하여 콘텐츠 등록을 유도.
  - **어뷰징 필터링:** 데이터 분석을 통해 광고성, 허위 리뷰를 필터링하는 시스템 구축.

- **② 취향저격 맞춤 코스 (AI Itinerary Planner):**

  - 사용자가 `#맛집탐방`, `#사진찍기좋은곳` 등 관심사 태그 선택.
  - **초기 기술:** 사용자가 선택한 태그와 장소의 태그를 매칭하는 **규칙 기반(Rule-based) 알고리즘**으로 시작.
  - **고도화 계획:** 향후 사용자 행동 데이터(찜, 검색, 체류 시간 등)를 활용한 협업 필터링 모델로 발전.

- **③ 로컬 밋업 (Connect with Locals):**

  - 검증된 현지인 가이드와 소규모 밋업을 연결하는 기능. (선택적 기능)
  - **안전 및 신뢰 장치:**
    - **다중 인증:** 휴대폰 본인인증, SNS 계정 연동을 통한 신원 확인.
    - **상호 평가 및 후기:** 만남 이후 상세한 후기 및 평점 시스템 의무화.
    - **UI/UX:** '인증 완료' 아이콘, '신고하기' 버튼, 안전 가이드라인 등 신뢰를 주는 시각적 장치 마련.

- **④ 실시간 현지 소식 피드 (Live Feed):**
  - 현지인들이 공유하는 실시간 로컬 이벤트, 플리마켓, 공연 정보 제공.
  - 사용자 위치 기반으로 주변 소식 푸시 알림 기능.

**5. 주요 도전과제 및 해결 전략**

- **콜드 스타트 (초기 콘텐츠 부족):**
  - **전략 1 (초기 기여자 모집):** 특정 1~2개 도시를 타겟으로, 해당 지역 거주 유학생/인플루언서/블로거를 '초기 기여자'로 섭외하여 양질의 초기 데이터를 확보 (소정의 리워드 제공).
  - **전략 2 (데이터 파트너십):** 지역 독립서점, 소규모 커뮤니티 등과 제휴하여 장소 정보를 제공받는 방안 모색.
- **경쟁 시장 (차별화):**
  - **전략:** 네이버/구글맵 등 거대 플랫폼과의 직접 경쟁을 피하고, '신뢰할 수 있는 현지인의 깊이 있는 이야기'라는 차별점에 집중. 우리는 '가장 많은 정보'가 아닌 '가장 진솔한 경험'을 제공하는 버티컬 서비스임을 강조.

**6. 수익 모델 (Business Model)**

- **Freemium 모델:** 핵심 기능 무료 제공. 프리미엄 구독자에게 무제한 맞춤 코스, 오프라인 지도 등 제공.
- **중개 수수료:** '로컬 밋업' 연결 시, 투어/체험 상품의 일부를 수수료로 확보.
- **B2B 솔루션:** 소상공인 대상 비즈니스 계정 제공. 앱 내에서 자연스럽게 전환을 유도하는 채널 마련.

**7. 기술 스택 및 개발 계획**

- **플랫폼:** Flutter 또는 React Native (iOS/Android 크로스플랫폼).
- **백엔드:** Python (FastAPI) 기반 API 서버.
- **데이터베이스:** PostgreSQL + PostGIS (위치 기반 서비스에 최적화).
- **클라우드:** AWS 또는 GCP.
- **외부 서비스 의존성 (예상):**
  - 지도: 네이버/카카오/구글맵 API
  - 인증: 휴대폰 본인인증 서비스, 소셜 로그인 API
  - 결제: PG사 연동 (향후)
- **개발 단계 (MVP):**
  - **1단계 (핵심 가치 검증):**
    - **기능:** 현지인 추천 스팟 등록/조회, 키워드/태그 기반 검색, 사용자 프로필.
    - **목표:** 타겟 도시 1곳에 집중하여, 사용자들이 정말 '이야기'가 담긴 장소를 추천하고 소비하는지 검증.
  - **2단계 (고도화):** 규칙 기반 AI 추천 코스, 실시간 피드 기능 추가.
  - **3단계 (확장):** 로컬 밋업 기능 도입, 수익 모델 적용, 서비스 지역 확장.

**8. 시장 진출 (Go-to-Market) 전략**

- **1단계 (거점 확보):** 국내 특정 도시(예: 연남동, 성수동 등 트렌드에 민감한 지역) 또는 특정 해외 도시(예: 방콕, 타이베이 등 자유여행객이 많은 곳) 1곳을 선정하여 집중 공략.
- **2단계 (커뮤니티 침투):** 타겟 도시의 여행/로컬 커뮤니티(온라인 카페, 페이스북 그룹 등)와 제휴 및 콘텐츠 확산.
- **3단계 (바이럴 마케팅):** 초기 사용자들이 등록한 매력적인 '스토리'를 콘텐츠로 가공하여 SNS 채널에 홍보. '우리 동네 찐맛집/찐장소 제보' 이벤트 등을 통해 사용자 참여 유도.
